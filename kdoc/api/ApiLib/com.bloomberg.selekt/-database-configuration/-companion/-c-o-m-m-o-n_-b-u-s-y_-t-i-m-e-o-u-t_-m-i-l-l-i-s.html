COMMON_BUSY_TIMEOUT_MILLIS

const val COMMON_BUSY_TIMEOUT_MILLIS: Int = 2_500

When using WAL, a timeout could occur if one connection is busy performing an auto-checkpoint operation. The busy timeout needs to be long enough to tolerate slow I/O write operations but not so long as to cause the application to hang indefinitely if there is a problem acquiring a database lock.
